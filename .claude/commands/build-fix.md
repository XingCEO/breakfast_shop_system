# /build-fix — 修復建置錯誤

## 流程

1. 執行建置指令
2. 按檔案與嚴重程度解析錯誤
3. 逐一修復每個錯誤

## 修復步驟（每個錯誤）

1. 顯示錯誤前後 5 行上下文
2. 說明問題原因
3. 提出修復方案
4. 實作修復
5. 重新建置
6. 確認修復成功

## 停止條件

出現以下情況時應暫停：
- 修復引入了新錯誤
- 同一錯誤嘗試 3 次仍未解決
- 使用者要求暫停

## 報告

完成後提供：
- 已修正的錯誤清單
- 剩餘的問題
- 新引入的問題（如有）

## 原則

**一次處理一個錯誤**，保持安全性與清晰度。

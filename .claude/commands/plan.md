# /plan — 功能規劃

使用 planner agent 來結構化實作工作。

## 執行流程

1. **重述需求** — 確認理解正確
2. **風險評估** — 識別潛在風險與阻礙
3. **分階段規劃** — 建立分階段實作策略
4. **等待確認** — 必須獲得使用者核准才能開始寫程式碼

## 使用時機

- 開發新功能
- 重大架構變更
- 複雜的重構作業
- 影響多個檔案或元件的修改
- 需求不明確時

## 規劃內容

請分析需求並提供：
- 清楚的需求摘要
- 分階段的實作步驟
- 元件相依性分析
- 風險評估與複雜度估算
- 明確的停止點，等待確認

## 重要原則

**在使用者明確確認計畫前，不要寫任何程式碼。**

使用者可以用以下方式回應：
- `好` / `proceed` / `yes` — 確認執行
- `修改: [變更內容]` — 調整計畫
- `換個方式: [替代方案]` — 使用不同方法

## 後續步驟

計畫確認後，可以接續：
- `/tdd` — 測試驅動開發
- `/build-fix` — 修復建置錯誤
- `/code-review` — 程式碼品質審查

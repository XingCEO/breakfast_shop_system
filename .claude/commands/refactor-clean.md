# /refactor-clean — 清理無用程式碼

## 流程

1. 掃描未使用的程式碼（exports、imports、變數、函式）
2. 依風險等級分類：
   - **SAFE**: 測試工具、明顯無用的程式碼
   - **CAUTION**: 元件、路由
   - **DANGER**: 設定檔、進入點
3. 只提議保守的移除候選項

## 安全協議

每次刪除前必須：
1. 執行完整測試套件
2. 確認測試通過
3. 套用變更
4. 重新執行測試
5. 如果測試失敗，立即回滾

## 報告

完成後提供：
- 發現的無用程式碼
- 已移除的項目
- 保留的項目（附原因）
- 測試結果摘要
